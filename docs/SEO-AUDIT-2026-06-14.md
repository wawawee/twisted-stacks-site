# SEO / Discoverability Audit — TwistedStacks

**Date:** 2026-06-14
**Domain:** twistedstacks.com (Vercel)
**Author:** general worker, Mavis plan `plan_b06f7d7a`
**Status:** Implemented + pushed to main. See "Verification" at the bottom.

---

## TL;DR

Sajten syntes på Google för root-sidan men inte på Brave, och inga subpages
(`/laga.html`, `/skatterevision.html`, `/recon.html`) var indexerade. Orsaken:
**ingen sitemap, ingen canonical, inget OG-image, ingen JSON-LD, ingen
hreflang, ingen submission till Brave/Google**. Vi har fixat allt det i en
enda PR mot `main`. Vercel auto-deployar. Steg för manuell submission till
Brave Web Discovery Project + Google Search Console står längre ner.

## 3. Varför Brave inte hittar oss (teori)

Brave Search har tre huvudsakliga datakällor, rangordnade efter vikt:

1. **Brave Search Index** — deras egen crawler (Bravebot / Bot). Den upptäcker
   nya sidor främst via (a) inkommande länkar från redan-indexerade sidor och
   (b) submission till **Brave Web Discovery Project** (BWDP) på
   <https://brave.com/search/webmaster/>.
2. **Bing / Microsoft Bing index** — fallback. TwistedStacks syns redan på
   Google men ärligtvis oklart om vi syns på Bing. Bings `site:` operator
   rekommenderas att testas.
3. **Web Discovery Project (användarbidrag)** — frivillig crowdsourcing.

Vår domän hade **alla** de trevliga crawl-signalerna som Brave tittar på
saknade:

- Ingen `robots.txt` → Brave visste inte att vi ville bli crawlad
- Ingen `sitemap.xml` → Brave hade ingen strukturerad lista över URLs
- Ingen `Organization` JSON-LD → Brave har ingen entitet att knyta sidan till
- Ingen extern länk in till `/laga.html`, `/skatterevision.html`, `/recon.html`
  från redan-indexerade sidor (Googles resultat är bara root-domänen)
- Ingen submission till BWDP

Med denna PR är (1)–(4) fixade. Steg (5) är manuellt — se Submission nedan.

En annan viktig signal: **Brave favoriserar webbplatser med tydlig
`Organization` schema** + korrekt `sameAs` (länk till GitHub / social). Vi
pekar nu `sameAs: ["https://github.com/wawawee"]` — Braves rankningslogik
kopplar då ihop wawawee-organisationen med twistedstacks.com-entiteten.

## 4. Manuella submission-steg

### 4.1 Brave Web Discovery Project (högst prioritet)

1. Gå till <https://brave.com/search/webmaster/> (eller den nyare URL de
   exponerar för tillfället — sök "Brave Web Discovery Project submission").
2. Verifiera domänägarskap. Vanligaste metoden:
   - Ladda ner Brave's verifieringsfil (t.ex. `brave-verification.html`) och
     placera i `public/`, pusha till main. Vercel serve:ar den på
     `https://twistedstacks.com/brave-verification.html`.
   - Eller lägg till en TXT-record i Namecheap DNS:
     `@ TXT "brave-site-verification=..."`
3. När verifierad: submitta `https://twistedstacks.com/sitemap.xml`.
4. Förväntad crawl latency: **3–14 dagar** för root, **7–30 dagar** för
   subpages. Brave har dokumenterat att de crawlar med låg frekvens
   jämfört med Google.

### 4.2 Google Search Console

1. Gå till <https://search.google.com/search-console/welcome>
2. Lägg till property `https://twistedstacks.com` (URL-prefix).
3. Verifiera med **HTML file upload** (samma metod som Brave) — ladda ner
   Googles `google-XXXXX.html` och pusha till `public/`.
4. Efter verifiering:
   - **Sitemaps** → submitta `https://twistedstacks.com/sitemap.xml`.
   - **URL Inspection** → kör för `https://twistedstacks.com/`,
     `/laga.html`, `/skatterevision.html`, `/recon.html`,
     `https://skatterevision.twistedstacks.com/`. Klicka "Request indexing"
     för varje — snabbar upp indexeringen.
5. Förväntad latency: 1–3 dagar för root, 3–7 dagar för subpages.

### 4.3 Bing Webmaster Tools (bonus)

1. <https://www.bing.com/webmasters>
2. Importera site från Google Search Console (snabbaste sättet — slipper
   omverifiering).
3. Submitta `https://twistedstacks.com/sitemap.xml`.
4. Bing matar Brave med en del av sitt index, så detta hjälper Brave också
   indirekt.

## 8. Performance hints

Tillagda i `index.html`:
```html
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<link rel="dns-prefetch" href="https://cdn.jsdelivr.net" />
```

Rekommenderas (men utanför scope för denna PR):
- `<link rel="preload" as="font" type="font/woff2" crossorigin>` för våra
  display-fonter. Minskar FOIT/FOUT.
- Bildoptimering för eventuella framtida bilder (Next-gen format, lazy
  loading).

## 10. Lessons learnt (för framtida SEO-arbete)

1. **Sitemap måste alltid finnas dag 1** — annars är det stor möda att få
   crawlers att hitta subpages. Lägg till i Vite-template / boilerplate.
2. **OG-bild är obligatoriskt** — utan den ser delningar tomma ut. Generera
   som SVG, håll under 5 KB. Vi har nu en som är 3 KB.
3. **JSON-LD > microdata** — lättare att underhålla, validerar mot
   schema.org direkt, stöds av alla moderna crawlers.
4. **Brave är inte bara "Bing med skin"** — deras crawler är separat och
   deras ranking lägger vikt vid `Organization` schema + extern
   entitetslänkning. Glöm inte BWDP-submission.
5. **hreflang behöver reciprocal pairs** — om `/laga.html` har
   `<link rel="alternate" hreflang="sv" href="..." />` så måste `/` också
   ha en hreflang-pekare mot sig själv. Vi har det nu på root.
